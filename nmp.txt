Nmap (Scanning Tool) 
A tool used to discover hosts and open ports in a network.  
Helps in security auditing and vulnerability detection.  
Can perform port scan, OS detection, and service version detection.  
 
1. Full Question 
Using Nmap perform Ping Scan (determine which hosts are up). Also show how to 
scan a single IP. 
 
    Important Terms 
 Nmap → Network scanning tool 
 Ping Scan (-sn) → Checks which hosts are active (no port scan) 
 Host → Device in network 
 Single IP Scan → Scan one specific system 
 
      Demo 
     Ping scan: 
nmap -sn 192.168.1.0/24 
     Single IP: 
nmap 192.168.1.1 
 
     Output Example 
Nmap scan report for 192.168.1.1 
Host is up 
 
PORT   STATE SERVICE 
80/tcp open  http 
22/tcp open  ssh 

Here I used Nmap ping scan to identify active hosts in the network. The -sn option 
skips port scanning and only checks if hosts are up. Then I scanned a single IP to find its 
open ports and running services.

2. Full Question 
Using Nmap, scan a range of IPs. Also scan an entire subnet. 
Important Terms 
Range of IPs → Specific group (e.g., 1–50) 
Subnet → Entire network (e.g., /24) 

Demo 
nmap 192.168.1.1-50 
nmap 192.168.1.0/24 
Output 
192.168.1.1 → open ports: 80, 22   
192.168.1.5 → open ports: 443   

I scanned a range of IP addresses and an entire subnet using Nmap. This helps in 
discovering all active systems and their open ports within a network.

3. Full Question 
Using Nmap, scan top 1000 ports (default). Also show how to detect service 
version. 
Important Terms 
Top 1000 Ports → Most commonly used ports 
Service Version Detection (-sV) → Finds software version running 
Output 
Ports + service + version 
Demo 
nmap -sV 192.168.1.1 
Output Example 
80/tcp open  http Apache 2.4.41 
22/tcp open  ssh OpenSSH 7.9 

Here I scanned the target using Nmap’s default top 1000 ports. Using the -sV option, I 
also detected the version of services running on those ports, which is useful for 
vulnerability assessment.

4. Full Question 
Using Nmap show OS detection. Also demonstrate how to perform Aggressive 
scan. 
Important Terms 
OS Detection (-O) → Identifies operating system 
Aggressive Scan (-A) → Enables OS detection + version + scripts + traceroute 
Output 
OS info + services + extra details 
Demo 
nmap -O 192.168.1.1 
nmap -A 192.168.1.1 
Output Example 
OS: Linux 5.x 
Service: Apache HTTP Server 
Network Distance: 2 hops 

I used Nmap to detect the operating system of the target using the -O option. Then I 
performed an aggressive scan using -A, which provides detailed information including 
OS, services, versions, and traceroute

5. Full Question 
Using Nmap, scan for vulnerabilities. Also demonstrate how to save results to a 
text file. 
Important Terms 
Vulnerability Scan → Finds security weaknesses 
Nmap Scripts (--script vuln) → Predefined scripts for vulnerabilities 
-oN → Save output in normal format 
Output 
List of vulnerabilities 
Saved file 
Demo 
nmap --script vuln 192.168.1.1 -oN result.txt 
Output Example 
VULNERABLE: 
OpenSSL Heartbleed vulnerability 
File created: 
result.txt 

In this experiment, I used Nmap scripting engine to scan for vulnerabilities. The --script 
vuln option checks for known security issues. The results are also saved into a file using -oN, which can be used for reporting.
