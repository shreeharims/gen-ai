UNION-based SQL Injection 
UNION SELECT: This is the defining feature of UNION-based SQL 
injection. It allows an attacker to append results from another SELECT 
statement to the original query. 
Column Matching: The UNION SELECT must match the number and type 
of columns from the original query. In your example, the attacker selects email, 
passwd, and two dummy values (123, 123) to match what the application 
expects. 
Data Extraction: The payload is designed to extract sensitive data (email, 
passwd) from the userdb table. 
Commenting Out (-- or #): This ensures that the rest of the original SQL 
query is ignored, preventing syntax errors or interference.

predictable structure XXX:YYY:… etc., and it probably 
contains information it shouldn’t. Which field of the 
decoded string could an attacker change to give himself a 
discount on an item’s price? 
After getting the CartID value , and url-decoding it 
using https://www.urldecoder.org 
CartID=1527258326%3A1%3A11.5%3A1000;

1527258326:1:11.5:1000 
Field 1 : 1527258326 is a unix timestamp 
Field 2 : “Useless” 
Field 3 : 11.5 is the price 
Field 4 : 1000 is the item id. 
So the attacker needs to edit the price field which is the third order.
